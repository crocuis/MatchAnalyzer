# 데일리 베팅 추천 화면 설계

## 목적

데일리 베팅 추천 화면은 기존 매치/리그 대시보드와 역할을 분리한 실행형 화면이다. 사용자는 리그별 경기 목록을 훑는 대신, 오늘 실제로 검토할 추천 5~10건을 한 번에 확인한다.

이 화면은 승무패, 핸디캡, 언더/오버 추천을 같은 기준으로 비교하게 하되, 미래 수익을 보장하는 문구를 사용하지 않는다. UI는 백테스트 기준, 현재 신뢰도, 기대값, 시장 가격, 보류 사유를 함께 보여준다.

## 진입 동선

대시보드 첫 화면에는 `Daily Picks`로 가는 명확한 진입점을 둔다.

- 헤더 우측에 항상 보이는 `Daily Picks` 버튼을 둔다.
- 첫 화면 상단에는 `오늘의 추천 보드` 요약 영역을 둔다.
- 요약 영역은 추천 수, 업데이트 시각, 대상 시장, 백테스트 기준을 간결하게 보여준다.
- 리그 요약에서는 `리그 추천 보기` 링크로 Daily Picks를 해당 리그 필터 상태로 연다.
- 매치 카드에 추천 배지가 있으면 해당 매치로 필터된 Daily Picks 또는 기존 매치 상세로 진입할 수 있게 한다.

Daily Picks 화면에서 각 추천 항목을 누르면 기존 매치 상세/리포트로 drill-down한다. 추천 리스트는 실행 목록이고, 상세 리포트는 근거 확인과 사후 검증 화면이다.

## 화면 구조

Daily Picks는 별도 화면으로 구성한다.

상단 영역:
- 오늘 추천 수와 추천 생성 시각
- 백테스트 기준 요약: 적중률 목표, ROI 목표, 하루 추천 수 제한
- 빠른 필터: 전체, 승무패, 핸디캡, 언더/오버, 리그
- 보류 포함 토글: 기본값은 추천 항목만 표시

본문 영역:
- 추천 항목을 우선순위순으로 세로 리스트 또는 밀도 높은 카드 리스트로 보여준다.
- 데스크톱에서는 좌측 리스트와 우측 상세 패널을 둘 수 있다.
- 모바일에서는 추천 리스트를 먼저 보여주고, 항목 선택 시 상세 바텀시트나 상세 화면으로 전환한다.

하단 또는 보조 영역:
- 보류된 후보 요약을 선택적으로 보여준다.
- 보류 사유는 `신뢰도 부족`, `시장 데이터 부족`, `EV 부족`, `라인업 미확정`처럼 짧은 상태로 표기한다.

## 추천 항목 카드

추천 카드는 실행 판단에 필요한 정보를 한 번에 읽게 만든다.

필수 정보:
- 경기: 홈팀, 원정팀, 킥오프 시각, 리그
- 추천 시장: 승무패, 핸디캡, 언더/오버
- 추천 선택지: 예: `HOME`, `Inter -0.5`, `Under 2.5`
- 신뢰도
- 기대값 또는 edge
- 시장 가격과 모델 확률
- 추천 상태: 추천, 보류, 결과 대기, 적중, 미적중

보조 정보:
- 소스 합의도
- 데이터 품질 배지: 라인업 확인, 시장 데이터 있음, 동기화 누락
- 추천 사유 1~2개
- 리포트 열기 버튼

카드 색상은 수익을 암시하는 과장된 색보다 상태 구분에 집중한다. 추천은 차분한 강조색, 보류는 중립색, 적중/미적중은 결과 확인 이후에만 성공/위험색을 사용한다.

## 데이터 흐름

Daily Picks 화면은 기존 매치 API에 섞기보다 전용 응답을 받는 것이 좋다.

권장 API 형태:
- `GET /api/daily-picks?date=YYYY-MM-DD`
- 선택 필터: `leagueId`, `marketFamily`, `includeHeld`
- 응답은 추천 리스트, 생성 시각, 백테스트 기준, 데이터 커버리지 요약을 포함한다.

추천 항목은 기존 prediction/match 식별자를 포함해야 한다. 그래야 추천 카드에서 기존 매치 상세와 Full Report를 재사용할 수 있다.

## 상태 처리

로딩:
- 오늘 추천 보드를 skeleton으로 보여준다.
- 추천 생성 시각은 데이터가 있을 때만 표시한다.

빈 상태:
- 추천 조건을 만족하는 항목이 없으면 `오늘은 추천 조건을 통과한 항목이 없습니다`로 표시한다.
- 보류 후보가 있으면 `보류 후보 보기`를 제공한다.

부분 데이터:
- 승무패만 있고 핸디캡/언더오버 데이터가 없으면 시장별 커버리지 요약에 표시한다.
- 누락을 숨기지 말고, 추천 수가 적은 이유를 설명한다.

오류:
- API 오류는 대시보드 전체 오류와 분리한다.
- Daily Picks만 실패하면 기존 리그 대시보드는 계속 사용할 수 있어야 한다.

## 테스트 기준

프론트엔드 테스트:
- 헤더의 `Daily Picks` 진입점 렌더링
- 오늘 추천 보드 CTA 렌더링
- 추천 카드의 시장 유형별 표시: 승무패, 핸디캡, 언더/오버
- 보류 항목 토글
- 추천 항목 클릭 시 매치 상세/리포트 연결

API 테스트:
- 날짜별 추천 목록 응답
- 시장 유형 필터
- 리그 필터
- 추천 5~10건 제한 표시
- 데이터가 부족할 때 커버리지/빈 상태 응답

회귀 테스트:
- 기존 리그 탭과 매치 카드 동작 유지
- 기존 매치 상세 모달/Full Report 진입 유지
- 기존 prediction summary 계산 유지

## 구현 범위

1차 구현 범위:
- 대시보드 헤더의 `Daily Picks` 버튼
- 첫 화면의 오늘 추천 보드 요약 CTA
- 별도 Daily Picks 화면
- 추천 카드 리스트
- 기존 매치 상세/리포트로 연결
- 기본 필터: 시장 유형, 리그, 보류 포함

1차 제외 범위:
- 실제 베팅 실행 또는 외부 베팅 사이트 연동
- 금액 자동 산정
- 푸시 알림
- 실시간 배당 갱신 애니메이션
- 미래 수익 보장 문구

## 남은 리스크

현재 핸디캡과 언더/오버는 실제 historical coverage가 부족할 수 있다. UI는 추천 수와 커버리지 부족을 솔직하게 보여줘야 한다. 추천 성과 수치는 백테스트 기준으로 표기하고, 실제 미래 수익 보장처럼 보이지 않도록 문구를 제한한다.
